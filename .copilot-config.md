# GitHub Copilot Automation Configuration

## 🤖 Copilot Automation Rules

Este archivo define las automaciones que GitHub Copilot debe ejecutar automáticamente en PRs y issues.

### PR Review Automation

**Triggers:**
- `review: auto` → Genera automáticamente sugerencias de review
- `linting: check` → Verifica linting automáticamente
- `security: scan` → Ejecuta security scan
- `docs: update` → Sugiere actualizar documentación

**Response Format:**
```
@copilot review: auto
```

### Issue Automation

**Triggers:**
- `bug` → Label automático, prioridad alta
- `feature` → Label automático, prioridad media
- `help wanted` → Label automático
- `security` → Label automático, prioridad crítica

### Automated Actions

#### 1. PR Labeling (Automático)

```yaml
Rules:
  - Si PR tiene cambios en `backend/` → `backend` label
  - Si PR tiene cambios en `frontend/` → `frontend` label
  - Si PR tiene cambios en linting/config → `chore` label
  - Si PR tiene cambios en docs → `documentation` label
  - Si PR cierra un issue → `closes-issue` label
```

#### 2. PR Reviewer Assignment (Automático)

```yaml
Rules:
  - Backend changes → Asignar revisores especializados en backend
  - Frontend changes → Asignar revisores especializados en frontend
  - Security changes → Asignar al security team
```

#### 3. Auto-Merge Rules (Automático)

```yaml
Conditions para auto-merge:
  - All checks passing ✅
  - Approved by at least 1 reviewer ✅
  - No conflicts ✅
  - No requested changes ✅
  - Mergeable state = true ✅

Exclusions (Requieren review manual):
  - PRs de Dependabot con updates MAJOR
  - PRs que cambien dependencias críticas
  - PRs que cambien configuración de seguridad
  - PRs que afecten el deploy
```

#### 4. Auto-Update Rules (Automático)

```yaml
Auto-update cuando:
  - Base branch recibe cambios importantes
  - Hay conflictos que pueden resolverse automáticamente
  - CI pasa en el PR actual
```

#### 5. Notifications (Automático)

```yaml
Cuando notificar:
  - Security vulnerability encontrada
  - Performance degradation detectado
  - Breaking change detectado
  - Coverage baja en cambios
```

### Copilot Commands

#### Global Commands
```
@copilot help                    # Mostrar ayuda
@copilot summary                 # Resumen del repo
@copilot status                  # Estado actual de PRs/Issues
```

#### PR Commands
```
@copilot review: auto            # Revisar automáticamente
@copilot approve                 # Aprobar el PR
@copilot suggest: improvements   # Sugerir mejoras
@copilot check: security         # Verificar seguridad
@copilot check: performance      # Verificar performance
@copilot generate: tests         # Generar tests
```

#### Issue Commands
```
@copilot convert: pr             # Convertir issue a PR
@copilot suggest: solution       # Sugerir solución
@copilot assign                  # Asignar issue
@copilot estimate                # Estimar esfuerzo
```

#### Workflow Commands
```
@copilot run: tests              # Ejecutar tests
@copilot run: lint               # Ejecutar linting
@copilot run: security-scan      # Ejecutar security scan
@copilot deploy: staging         # Deploy a staging
@copilot deploy: production      # Deploy a production
```

### Copilot Behaviors

#### Auto-Analysis (Siempre Activo)
- ✅ Analiza cada PR automáticamente
- ✅ Genera sugerencias de mejora
- ✅ Detecta errores comunes
- ✅ Verifica coverage
- ✅ Chequea seguridad

#### Auto-Comments (Configurable)

```yaml
Sugerencias automáticas para:
  - Missing error handling
  - Performance improvements
  - Security vulnerabilities
  - Code duplication
  - Documentation gaps
  - Test coverage gaps
```

#### Auto-Suggestions (Configurable)

```yaml
Sugiere automáticamente:
  - Refactoring opportunities
  - Design pattern improvements
  - Best practices
  - Performance optimizations
  - Security hardening
```

### Security & Safety Rules

```yaml
Never Auto-Merge:
  - Security-related changes
  - Database migration changes
  - Infrastructure changes
  - Breaking API changes
  - Major dependency upgrades

Require Manual Review:
  - Changes to auth/security
  - Changes to database schema
  - Changes to deployment config
  - Changes to CI/CD workflows
  - Changes to sensitive data handling
```

### Escalation Rules

Escalar a humanos cuando:
- ❌ Conflicto de merge no resolvible automáticamente
- ❌ Tests fallando después de merge
- ❌ Security vulnerability crítica encontrada
- ❌ Performance degradation detectado
- ❌ Breaking change detectado
- ❌ Ambigüedad en requirements

### Metrics & Reporting

Copilot debe trackear:

- ✅ Security issue rate

Generar reporte semanal con:
